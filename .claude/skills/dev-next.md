# /dev-next

의존성 그래프를 분석하여 다음에 구현할 스펙을 추천합니다.

## 사용법

```
/dev-next
```

## 워크플로우

1. `.sdd/specs/` 디렉토리의 모든 스펙을 로드합니다
2. 각 스펙의 `depends_on` 필드를 분석합니다
3. 의존성 그래프를 구성합니다
4. 구현되지 않은 스펙 중 의존성이 모두 충족된 것을 찾습니다
5. 우선순위에 따라 다음 구현할 스펙을 추천합니다

## 출력 형식

```
📋 다음 구현 추천:

1. auth/logout (의존성 없음)
   설명: 로그아웃 기능
   예상 난이도: 낮음

2. user/profile (auth/login 의존)
   설명: 사용자 프로필 조회
   예상 난이도: 중간

💡 /dev-implement auth/logout 으로 구현을 시작하세요.
```

## 참고사항

- 의존성이 충족되지 않은 스펙은 추천하지 않습니다
- 복잡도가 낮은 스펙을 우선 추천합니다
- `.sdd/.tasks/`의 진행 상황을 참고합니다
