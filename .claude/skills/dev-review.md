# /dev-review

구현된 코드가 스펙을 충족하는지 검증하고 코드 품질을 검토합니다.

## 사용법

```
/dev-review [spec-id]
/dev-review --all
```

## 워크플로우

1. 스펙 파일을 로드합니다
2. 관련 구현 파일을 찾습니다
3. 다음 항목을 검증합니다:
   - 스펙의 시나리오가 테스트로 구현되었는가
   - 입출력 계약이 준수되는가
   - 에러 처리가 적절한가
   - 타입 안전성이 확보되었는가
4. 코드 품질을 분석합니다:
   - 네이밍 규칙 준수
   - 주석/문서화
   - 코드 복잡도
5. 리뷰 결과를 출력합니다

## 출력 형식

```
📝 스펙 리뷰: auth/login

✅ 시나리오 충족: 3/3
✅ 계약 준수: 입력/출력 타입 일치
⚠️ 경고: 에러 메시지 하드코딩
❌ 문제: 테스트 커버리지 부족 (60%)

💡 개선 제안:
1. 에러 메시지를 상수로 분리하세요
2. 경계값 테스트를 추가하세요
```

## 참고사항

- `code-reviewer` 에이전트를 활용합니다
- 심각한 문제가 있으면 구현을 중단합니다
- 모든 시나리오가 테스트되어야 합니다
